# Issue Guide

Use issue templates for consistent triage.

## Before opening an issue

1. Confirm reproduction steps.
2. Include exact command and output.
3. Include environment details (OS, shell, Rust/Node versions).
4. State expected vs actual behavior.

## Labels

- `bug`
- `feature`
- `docs`
- `good-first-issue`
- `needs-repro`

## Fast triage recommendations

- Attach minimal diff fixture for scoring bugs.
- Include sample commit message and changed entities when reporting alignment mismatches.
